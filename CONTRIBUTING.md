# Contributing to CineSense

First off, thank you for considering contributing to CineSense! It's people like you that make CineSense such a great tool.

## Where do I go from here?

If you've noticed a bug or have a question, [search the issue tracker](https://github.com/chirag127/CineSense-Movie-Recommendation-Streamlit-Web-App/issues) to see if someone else has already created a ticket. If not, go ahead and [create a new issue](https://github.com/chirag127/CineSense-Movie-Recommendation-Streamlit-Web-App/issues/new/choose)!

## Fork & create a branch

If you're looking to contribute to the codebase, the first step is to fork this repository. Once you've done that, you can create a new branch for your feature or bug fix.

```bash
git checkout -b my-new-feature
```

## Get the code running

Once you've cloned your fork and created a new branch, you'll need to get the project running. The instructions for this are in the [README.md](README.md) file.

## Making your changes

Once you've got the code running, you can make your changes. We've included a `ruff.toml` file to help you keep your code style consistent with the rest of the project.

## Submitting a pull request

When you're ready to submit a pull request, please make sure to do the following:

-   Run `ruff check .` to make sure your code is linted.
-   Write a clear, descriptive title and summary for your pull request.
-   Reference any relevant issues in your pull request description.

Thank you for your contribution!
